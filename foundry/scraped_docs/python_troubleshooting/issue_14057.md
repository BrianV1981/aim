# Q: Uncaught exceptions in asyncio tasks are silently ignored
**Source:** https://github.com/pytest-dev/pytest/issues/14057

## The Problem / Request
This code executed outside of pytest
```python
import asyncio

async def error():
    raise Exception

async def stop_loop(loop):
    await asyncio.sleep(3)
    loop.stop()

with asyncio.Runner() as runner:
    loop = runner.get_loop()
    loop.create_task(error())
    loop.create_task(stop_loop(loop))
    loop.run_forever()
```
will print
```
Task exception was never retrieved
future: <Task finished name='Task-1' coro=<error() done, defined at ...\asyncio_exception.py:3> exception=Exception()>
Traceback (most recent call last):
  File "...\asyncio_exception.py", line 4, in error
    raise Exception
Exception
```
But when executed within pytest
```python
import asyncio

def test_asyncio_error():
    async def error():
        raise Exception

    async def stop_loop(loop):
        await asyncio.sleep(3)
        loop.stop()

    with asyncio.Runner() as runner:
        loop = runner.get_loop()
        loop.create_task(error())
        loop.create_task(stop_loop(loop))
        loop.run_forever()
```
Not only does this not fail, but it doesn't print anything at all! So there is no indication of anything going wrong.

pytest is run with `addopts = --tb=short --capture=no -p no:warnings`

pytest 8.4.2, Windows 11

## The Solution / Discussion
### Response 1
Figured out this is because asyncio output is via `logging.error` https://github.com/python/cpython/blob/e25083abeefa48aad12bb05e8c8ed25e83c4d249/Lib/asyncio/base_events.py#L1875, and pytest is capturing the log output

Nevertheless, this is quite concerning as default behavior

### Response 2
It is unreasonable to expect core pytest to handle all user errors

When managing delicate topics like async tasks its fair to expect some diligence with the user

If you roll a own async thing handle the building blocks right
