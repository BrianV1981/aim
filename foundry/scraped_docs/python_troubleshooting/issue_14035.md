# Q: Monkeypatch. Multiprocessing. Mac OS
**Source:** https://github.com/pytest-dev/pytest/issues/14035

## The Problem / Request
Monkeypatch doesnt work if you are trying to test multiprocess func on Mac OS. Mac OS using spawn as default method of creating child process (except fork in classic linux), so its fully ignore memory patches as monkeypatch and there is no any alert about it.

It will be cool either add note to docs or set "fork" as a default multiprocess method while tests running.

## The Solution / Discussion
### Response 1
This doesn't necessarily sound related to pytest. Do you have a reproducer?

### Response 2
closing as out of scope - anything multiprocessing related is not safe to monkeypatch - design for test instrumentation is highly recommended as non-fragile alternative
