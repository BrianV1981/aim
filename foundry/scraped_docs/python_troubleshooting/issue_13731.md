# Q: disabling capture in cli doesn't work
**Source:** https://github.com/pytest-dev/pytest/issues/13731

## The Problem / Request
Using flag `--capture=no` or `-s` should disable capturing.

my venv:

```
❯ pip list
Package   Version
--------- -------
iniconfig 2.1.0
packaging 25.0
pip       24.0
pluggy    1.6.0
Pygments  2.19.2
pytest    8.4.2
❯ python3 --version
Python 3.12.3
```

my OS:

```
Ubuntu 24.04 on WSL
```

the test:

```python
def test_capsys(capsys):
    print('Hello World!')
    captured = capsys.readouterr()
    assert captured.out == 'Hello World!\n'
    assert captured.err == ''
```

To reproduce it, run `pytest --capture=no test.py` or `pytest -s test.py`. The test should fail as capture is disabled but it still pass.

## The Solution / Discussion
### Response 1
The capture fixtures enforce specific capture for specific tests 

So it ought to fail 

We should perhaps add a note/warning to the output 

### Response 2
I think it'd be sufficient to document that using a capture fixture takes priority over `--capture=no` in [the tutorial](https://docs.pytest.org/en/stable/how-to/capture-stdout-stderr.html).
