# Q: pyproject.toml [tool.pytest] env_files no longer used as of 9.0.0
**Source:** https://github.com/pytest-dev/pytest/issues/14043

## The Problem / Request
## Bug Description

I was previously using `pytest`, `python-dotenv`, and a `pyproject.toml` configuration set to load a file of environmental variables into Pytest to create a secondary environment for testing. As of 9.0.0, this no longer functions.

```toml
[tool.pytest]
addopts = ["--no-header", "-vv", "--color=yes"]
testpaths = ["tests"]
env_files = [".env.test"]
```
This worked in <9.0.0. As of 9.0.0, running tests throws this error:
```
PytestConfigWarning: Unknown config option: env_files
```
Please note that this is _without_ using pytest-dotenv. Installing pytest-dotenv does not resolve the issue.

I was not able to find any documentation specifically mentioning env_files as a native configuration variable in 8.X.X, nor was I able to find mention of its removal in the 9.0.0 changelog. Nonetheless, it was working before the update so I'm a bit stuck.

## The Solution / Discussion
### Response 1
Pytest before 9.0 didn't support tools pytest in toml configuration files

The referred plugin is abandoned and has dozens of bugs



### Response 2
I'm mad I didn't catch it, but you're right—the problem was twofold: I was missing `pytest-dotenv`, which I didn't include in testing the problem exactly because it was abandoned (although I did find it buried in the lock file of my original repo). But pytest-dotenv was also not working to fix the problem because I didn't have py*thon*-dotenv installed either and it was silently failing on that.

So yeah! All sorted, thank you.
