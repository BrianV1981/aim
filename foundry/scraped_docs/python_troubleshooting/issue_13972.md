# Q: 9.0.0 outputs control characters with '---color=no'
**Source:** https://github.com/pytest-dev/pytest/issues/13972

## The Problem / Request
When running pytest >=9.0.0 in emacs compilation mode I get:

```
test/lib/logging_filters_test.py ^[]9;4;1;36^[\.^[]9;4;1;36^[\.          [ 36%]
```

Adding  `--color=no` has no effect.

Pre 9.0.0 I get

```
test/lib/logging_filters_test.py ..                       [ 36%]
```

Adding  `--color=no` is not required.

### Response 2
If the problem is the terminal progress, then it will be resolved by #13971. If you're looking for a workaround until the next point release, you can use `-p no:terminalprogress`.

Although we should test that the TERM is not dumb as well, this way it won't have this issue even if explicitly enabled. I forgot about that, so thanks for reporting. I will add that to #13971.

### Response 3
