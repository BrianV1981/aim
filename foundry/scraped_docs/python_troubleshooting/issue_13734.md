# Q: IndexError if all post-cut traceback frames are hidden
**Source:** https://github.com/pytest-dev/pytest/issues/13734

## The Problem / Request

```python
import pytest

@pytest.mark.trio
async def test_xfail():
    pytest.xfail()
```

## The Solution / Discussion
### Response 1
So the problem arises in

https://github.com/pytest-dev/pytest/blob/6eb7609c7d738da904de5c8b0b7c9c2a80ddf3e0/src/_pytest/_code/code.py#L1191-L1198

when `filter_excinfo_traceback` returns an empty traceback - which in this case happens because pytest-trio is liberal with using `__tracebackhide__ = True`. I'm not entirely sure if that's a bug in pytest-trio, but pytest should handle this gracefully. It appears when I wrote that chunk I should've looked closer at some of the logic from earlier in the file and either done something like

https://github.com/pytest-dev/pytest/blob/6eb7609c7d738da904de5c8b0b7c9c2a80ddf3e0/src/_pytest/_code/code.py#L1113-L1125

or

https://github.com/pytest-dev/pytest/blob/6eb7609c7d738da904de5c8b0b7c9c2a80ddf3e0/src/_pytest/_code/code.py#L743-L753

it appears that ReprTracebackNative does have support for `extraline` - ~~so perhaps the logic in `getrepr` should be updated as well to also output a line about all traceback entries being hidden.~~
EDIT: filtering is disabled in `getrepr`, so it's not relevant.
