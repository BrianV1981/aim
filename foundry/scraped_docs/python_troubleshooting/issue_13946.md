## The Solution / Discussion
### Response 1
`config.inicfg` is a private implementation detail of pytest, so we don't ensure stability when accessing or modifying it.

Can you describe why you want to set `xfail_strict` from a hook instead of the usual way?

### Response 2
The reason for setting it in the hook is that the python test files are copied to other machines where they run in a completely different directory structure, i.e. working with configuration files in the source tree, will be difficult to mirror onto the destination. In contrast the hook is available in both cases. 

I was not aware `inicfg` was private, I would have expected a '_' prefix in that case.

### Response 3
Perhaps we can consider adding support for setting configuration options after the fact, though I'm not sure it's a good idea, as it could easily break various expectations. I think we are unlikely to support it.

I do not entirely follow your setup so it's hard to suggest an alternative, but have you considered using pytest-xdist [remote support](https://pytest-xdist.readthedocs.io/en/stable/remote.html)?

---

In any case, if you just want a quick fix, while still relying on private implementation details that might break in future versions, you can update your code to this:

```py
from _pytest.config.findpaths import ConfigValue

def pytest_configure(config):
    config.inicfg["xfail_strict"] = ConfigValue(True, origin="override", mode="toml")
```
