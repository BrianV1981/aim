# Q: The pytest.exe runner/wrapper has some incompatibility with uv which breaks history in pdb on windows
**Source:** https://github.com/pytest-dev/pytest/issues/14052

## The Problem / Request
Running pytest directly through uv breaks history in pdb on windows. i.e.
```
➜ cat src/shell_pytest/test_thingy.py -p
def test_my_test():
    breakpoint()
```
`uv run pytest` drops you straight into a debugger. On linux everything is good on windows history is broken.

This is quite annoying because pdb without history is very cumbersome! I'm not sure if this is easiest to fix on the uv side of things or in the `pytest.exe` wrapper so I've made issue in both. The uv issue is here:
https://github.com/astral-sh/uv/issues/17174
That issue also contains some more detail if anyone is interested in picking this up.

## The Solution / Discussion
### Response 1
Sounds like libedit/readline differences in their standalone builds (a known source of surprises). I'll leave this here: https://gregoryszorc.com/docs/python-build-standalone/main/quirks.html

### Response 2
Apparently this will be fixed by https://github.com/python/cpython/pull/143083
