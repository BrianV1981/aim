# Q: [tool.pytest] as introduced in pytest 9.0.0 in pyproject.toml is not yet supported by SchemaStore
**Source:** https://github.com/pytest-dev/pytest/issues/13985

## The Problem / Request
Hello!

With pytest version 9.0.0 we can finally just write `[tool.pytest]` instead of `[tool.pytest.ini_options]` in a `pyproject.toml` file but I think there are quite a lot of people who use VS Code with this nice extension https://marketplace.visualstudio.com/items?itemName=tamasfe.even-better-toml which does not yet know about this change and will mark `[tool.pytest]` as an error.

This has to be added to SchemaStore, see https://github.com/SchemaStore/schemastore/issues/3564 so consider this just a kind of PSA and also a hint for people who came here via a websearch.
