# Bug report

## The Solution / Discussion
### Response 1
> This leads to issues during traverse. In our own Python wrapper this did lead to the relation between the dict and it's owning class not getting found

What is the issue? Ignoring the return value doesn't mean that the object wasn't visited. 

### Response 2
I didn't construct a test with the Cpython types but at least for our own inhouse Python wrapper not handling this did lead to
```cpp
    instance = m.DynamicAttr()
    instance.a = "test"
    assert instance in gc.get_referrers(instance.__dict__)
```
failing. This was fixed by returning the corresponding return code instead of always returning 0.

This is in line with the Py_VISIT macro which also returns on a visit result != 0.

### Response 3
For pybind11 I did create the fix as well (https://github.com/pybind/pybind11/pull/6032) but there it continues failing for now since the ci is still on 3.14.3 and so doesn't have the fixes from (https://github.com/python/cpython/issues/130327) yet.
