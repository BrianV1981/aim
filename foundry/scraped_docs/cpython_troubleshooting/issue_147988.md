# Q: Detect usage of uninitialized digits in `Objects/longobject.c` (`_PyLong_New()`)
**Source:** https://github.com/python/cpython/issues/147988

## The Problem / Request
The `_PyLong_New()` API does not initialize the digits array. Catching usage of uninitialized digits requires building Python with MSAN (`-fsanitize=memory`) which is not easy to use in practice.

## The Solution / Discussion
### Response 1
> I propose changing long_alloc() to initialize digits to a pattern to detect the usage of uninitialized digits when Python is built in debug mode.

This strategy is already used by `k_mul()`:

```c
    /* 1. Allocate result space. */
    ret = long_alloc(asize + bsize);
    if (ret == NULL) goto fail;
#ifdef Py_DEBUG
    /* Fill with trash, to catch reference to uninitialized digits. */
    memset(ret->long_value.ob_digit, 0xDF, _PyLong_DigitCount(ret) * sizeof(digit));
#endif
```

I propose to generalize it directly in `long_alloc()`.

### Response 2
I used the following change to disable the freelist in `long_alloc()` to help MSAN to detect uninitialized digits:

```diff
-        result = (PyLongObject *)_Py_FREELIST_POP(PyLongObject, ints);
+        //result = (PyLongObject *)_Py_FREELIST_POP(PyLongObject, ints);
```

Without this change, MSAN considers that the digit is properly initialized if an object is picked from the freelist, and so MSAN fails to detect the usage of uninitialized digits.
