# MEMORY.md — Durable Long-Term Memory (A.I.M.)

*Last Updated: 2026-03-26*

## 1) Identity + Mission
- **Operator:** Brian Vasquez.
- **Mode:** High-autonomy technical lead.
- **Style:** Blunt, direct, token-efficient.
- **Mission:** Ship A.I.M. as a portable memory operating layer for CLI agents: durable recall, low-noise handoff, zero hardcoded machine paths, zero accidental private-memory leakage.

## 2) Permanent Rules
- **Portability First:** All paths resolve from runtime root discovery. Never hardcode roots.
- **Local Runtime Integrity:** Scripts/hooks prefer the project `venv` for deterministic behavior.
- **Destructive Memory Hygiene:** Durable memory is pruned, not endlessly appended. Delete stale facts.
- **Privacy Boundary:** Never commit secrets, personal continuity, private archives, or machine config.
- **Clean Slate Support:** Wipe continuity, memory, and local history artifacts when requested.
- **Do Not Exit Mid-Handoff:** Session close requires pulse/tail or equivalent handoff artifacts.

## 3) Architecture
- **Control Plane:** `core/CONFIG.json` manages local settings (paths, models, providers, vault, safety).
- **Command Surface:** `scripts/aim_cli.py` (entrypoint), `scripts/aim_config.py` (providers/MCP), `scripts/aim_init.py` (bootstrap).
- **Forensic Store:** `archive/engram.db` (SQLite + fragment storage + FTS5 lexical search).
- **Retrieval Model:** Hybrid recall (semantic embeddings + FTS5 lexical search).
- **Transcript Funnel:** Raw JSON (`archive/raw/`) → fragmented (`src/indexer.py`) → Engram DB (chunked).
- **Continuity Layer:** `continuity/CURRENT_PULSE.md` and `continuity/FALLBACK_TAIL.md` for live handoff.
- **Memory Pipeline:** Hooks compress active work into hourly logs, checkpoint context, and preserve tactical fallbacks.
- **Sync Layer:** `archive/sync/` for Git-safe Engram exports. (Obsidian mirroring is optional).

## 4) Active Technical Stack
- **Primary Language:** Python.
- **Storage:** SQLite (`archive/engram.db`) with FTS5.
- **Embeddings:** Local Ollama by default (Google/OpenAI-compatible backends supported).
- **Reasoning Providers:** Configurable via cockpit (OpenRouter, Anthropic, Codex CLI, local Ollama, OpenAI-compatible). **Note:** Google Provider is currently unstable (Persistent 403 Forbidden errors). Default to alternative providers.
- **Secret Storage:** Keyring-backed vault access.
- **IDE Bridge:** MCP server support as a first-class integration path.

## 5) Stable Project Truths
- **A.I.M. is hook-driven:** Session hooks (start, pre-compress, before/after-tool, end) are core.
- **A.I.M. is retrieval-first:** Agents search durable memory; hallucination is prohibited.
- **A.I.M. is machine-portable:** Config auto-repairs path drift.
- **A.I.M. is repo-conscious:** Durable intelligence is shared; private operator residue is strictly isolated.
- **A.I.M. is layered:** Tactical continuity, hourly memory, and Engram storage are strictly separated.
